# Security Policy

## Supported versions

This project is currently in early development. Security fixes are applied to the main branch.

## Reporting a vulnerability

If you find a privacy or security issue, open a GitHub issue with a clear description and reproduction steps.

Do not include private personal files, credentials, API keys, resumes, contracts, financial records, or confidential business material in public issues.

## Privacy-sensitive design

The repository intentionally ignores the local `workspaces/` folder because that folder may contain private source notes, interview preparation, or personal research.

Before publishing a fork or example, check that it contains only public or synthetic information.
